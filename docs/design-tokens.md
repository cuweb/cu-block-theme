# Design tokens

CSS custom properties that drive every visual layer. Defined in [`src/css/tokens.scss`](../src/css/tokens.scss) and imported by both `styles.scss` and `editor.scss`.

> **Auto-generated.** The file's header reads "Auto-generated by component2block — do not edit manually." It's also `.stylelintignore`d. Source-of-truth changes happen in the generator, not here.

All tokens use the `--rds--` namespace.

## When to use which

| Need                                     | Use                                                            |
| ---------------------------------------- | -------------------------------------------------------------- |
| In custom SCSS / CSS                     | The `--rds--*` token                                           |
| In `theme.json` or block editor settings | The WordPress preset variable (`--wp--preset--color--*`, etc.) |

The two parallel sets carry the same values where they overlap (palette colors, spacing, fonts). Tokens additionally cover concepts not in the editor (feedback colors, font weights, line heights, border radii).

--rds--font-size-medium   --rds--font-size-large
--rds--font-size-x-large  --rds--font-size-2x-large

Heading sizes:

```
--rds--font-size-heading-small
--rds--font-size-heading-medium
--rds--font-size-heading-large
--rds--font-size-heading-x-large
--rds--font-size-heading-primary
```

All use `clamp()` for fluid scaling.

### Font weights

```
--rds--font-weight-light    (300)
--rds--font-weight-normal   (400)
--rds--font-weight-medium   (500)
--rds--font-weight-bold     (700)
```

> **Note.** Only weights 300, 400, and 600 are loaded as `@font-face` declarations in `theme-rds.json`. Setting `--rds--font-weight-medium` (500) or `--rds--font-weight-bold` (700) will trigger browser faux-bold synthesis.
